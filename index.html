how my retrieval systems work — select a stage

RSS / REST → Postgres

Multi-source feeds normalised into one canonical schema. Idempotent by design — ON CONFLICT DO NOTHING on the article URL, so re-runs never duplicate.

Hand-written recursive splitter

1000-char chunks, 150 overlap, breaking on natural boundaries (paragraph → sentence → word). Written by hand so every trade-off is tunable and explainable.

text-embedding-3-small · 1536-dim

Stored in pgvector next to the articles — transactional with the data, no extra vector-DB infra. HNSW index, cosine distance.

Hybrid: vector + full-text, fused by RRF

Semantic arm (cosine) + lexical arm (tsvector/GIN) fused by Reciprocal Rank Fusion (k=60) — ranks, not scores, so nothing needs normalising. Answers cite their sources.
